POLÍTICA DE CONTRIBUCIÓN
===============================================================================

IMPORTANTE: Este proyecto solo acepta contribuciones a través de GitHub Issues.

CONTRIBUCIONES ACEPTADAS:
✅ Reportes de bugs - Apreciamos y damos la bienvenida a reportes de errores
✅ Sugerencias de funcionalidades - Comparte tus ideas para mejoras
✅ Feedback de documentación - Ayúdanos a mejorar nuestra documentación
✅ Preguntas - Pregunta en GitHub Discussions

CONTRIBUCIONES NO ACEPTADAS:
❌ Pull Requests - Serán cerrados sin revisión
❌ Contribuciones de código - No se aceptan
❌ Forks con modificaciones - Por favor no redistribuyas versiones modificadas

CÓMO REPORTAR BUGS
===============================================================================

¡Apreciamos los reportes de bugs! Aquí está cómo enviar uno:

1. VERIFICAR ISSUES EXISTENTES
------------------------------
Antes de crear un nuevo issue, por favor verifica si ya ha sido reportado:
- Buscar en: https://github.com/FelipeFariasAlfaro/Judo-Framework/issues

2. CREAR UN NUEVO ISSUE
-----------------------
Ve a GitHub Issues y haz clic en "New Issue"

3. PROPORCIONAR INFORMACIÓN COMPLETA
------------------------------------
Usar la siguiente plantilla:

**Describir el bug**
Una descripción clara y concisa de cuál es el bug.

**Para Reproducir**
Pasos para reproducir el comportamiento:
1. Crear archivo feature con '...'
2. Ejecutar comando '...'
3. Ver error

**Comportamiento Esperado**
Lo que esperabas que pasara.

**Comportamiento Actual**
Lo que realmente pasó.

**Ambiente:**
- OS: [ej., Windows 10, macOS 13, Ubuntu 22.04]
- Versión de Python: [ej., 3.9.7]
- Versión de Judo Framework: [ej., 1.5.9.5]
- Versión de Behave: [ej., 1.2.6]

**Muestra de código**
# Código mínimo para reproducir el issue
# Incluir archivos feature, environment.py, etc.

**Mensaje de error/Stack trace**
Pegar el mensaje de error completo aquí

**Contexto adicional**
Cualquier otra información relevante, screenshots, etc.

===============================================================================
CÓMO SUGERIR FUNCIONALIDADES
===============================================================================

¿Tienes una idea para mejorar Judo Framework? ¡Nos encantaría escucharla!

1. VERIFICAR SUGERENCIAS EXISTENTES
-----------------------------------
- Buscar issues existentes con label "enhancement"

2. CREAR UN FEATURE REQUEST
---------------------------
Usar la siguiente plantilla:

**¿Tu feature request está relacionado con un problema?**
Una descripción clara y concisa de cuál es el problema.

**Describe la solución que te gustaría**
Una descripción clara y concisa de lo que quieres que pase.

**Describe alternativas que has considerado**
Una descripción clara y concisa de cualquier solución alternativa.

**Contexto adicional**
Agrega cualquier otro contexto o screenshots sobre el feature request aquí.

**Casos de uso**
Describe cómo usarías esta funcionalidad en tus proyectos.

===============================================================================
PROCESO DE REVISIÓN
===============================================================================

1. RECEPCIÓN DEL ISSUE
----------------------
- El equipo de CENTYC revisa todos los issues
- Respuesta inicial dentro de 48-72 horas
- Clasificación y etiquetado apropiado

2. INVESTIGACIÓN
----------------
- Reproducción del bug (si aplica)
- Análisis de viabilidad (para features)
- Estimación de esfuerzo y prioridad

3. DESARROLLO
-------------
- Implementación por el equipo de CENTYC
- Testing exhaustivo
- Documentación actualizada

4. RELEASE
----------
- Inclusión en próximo release
- Notificación al reportador
- Actualización del changelog
